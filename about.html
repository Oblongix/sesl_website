About SESL

Our Mission

To bring deterministic, transparent, explainable reasoning to automated decision systems.

Founders

Vince Powell has spent over 25 years working in businesses developing, delivering and using technology to grow businesses and make them more efficent. Unusually while holding senior business roles, Vince continues to be deeply engaged with the technology. Frustrated with the lack of transparency and explainability in AI systems, and concerned with the effort (and cost) involved in trying to get them to answer correctly. Vince founded SESL to create a solution that prioritises certainty, clarity and explainability in automated decision-making.

Roadmap

We are at the start of our journey. Having developed the core rules engine, the user interface (the CLI and the MCP server) and run over 10,000 tests we want to get the technology out to people to use and develop industry specific solutions.

SESL Language - Open Source with Commercial Licence

Our initial focus is to give people access ot the language. We don't plan to make any substantive changes to the engine or the language (other than bug fixes and small additions) in the short term, but do see value in having more tools to manage the rules and scenarios, and improving the presentation of the rule tracing and fact usage.

SESL MCP - Charge per use

The SESL MCP is available as a demonstration, the solution provides a direct connection between tools like chatGPT, Cluade, Gemini or others and SESL. From this you can both create rules directly from documents as well as run SESL models and get SESL to provide answers which can then be added to further analysis or reports. We plan to enhance the MCP with more features and capabilities over time.

Industry Template Modeals - Charge per use

We intend to offer partners the ability to publish there SESL templates on the Platform and receive revenue from their use by others. Think of this as a kind of app store.

SESL Platform - SaaS Licence (some free components)

Work has already started on the management platform for SESL. From the platform you will be able to create and test models (as you do locally using the CLI); create and configure, MCP servers; Allocate models to MCP servers; Manage a library of models and share them with others; Manage and monitor the running of models allowing models to be tracked and audited; and finaly manage users of the models; and as mentioned early we will also have a template store to both publish templates and for others to pay for their use.

SESL API - Charged per use

The SESL API will provide another alternative to run models. Intended for use in application where high volume and fast response is expected.
